# VSHB Innopolis

Используй этот skill как локальный пакет документов по Весенней школе бакалавров Innopolis 2026.

## Что внутри

```text

├── SKILL.md
├── references/
│   ├── logistics/

│   │   ├── 02-self-route-other-points.md

│       └── INDEX.md
└── assets/
    └── originals/
        ├── logistics/   # исходные PDF по логистике
        ├── program/     # исходные PDF по расписанию и чек-листу
        └── forms/       # исходные DOCX формы для заселения
```

## Быстрый маршрут по данным

- Если спрашивают, как добраться, где точка сбора, какие автобусы/трансферы — сначала открой `references/logistics/INDEX.md`, потом нужный файл.
- Если спрашивают про расписание по дням, время, треки, аудитории, заезд/отбой — открой `references/program/schedule.md`.
- Если спрашивают, что взять с собой или какие документы нужны для заселения — открой `references/program/checklist.md`.
- Если спрашивают про формы, заполнение или нужно отдать оригинал файла — открой `references/forms/INDEX.md`, затем используй файлы из `assets/originals/forms/`.

## Правила работы

1. Сначала использовать обработанные Markdown-файлы из `references/`.
2. Открывать оригиналы из `assets/originals/` только если нужна точная проверка, повторная конвертация или работа с формой.
3. Не придумывать данные, которых нет в документах.
4. Если вопрос про конкретный день, отвечать по соответствующему фрагменту расписания, а не по всему документу целиком.
5. Если просят заполнить форму, сначала прочитать сам DOCX/PDF, затем помогать с полями.

## Полезные Telegram-боты и ссылки

### @InnoHelpBot

Используй для отслеживания автобусов и связанных подсказок по транспорту.

Дополнительно:

- расписание автобусов: https://innopolis.com/ru/city/bus-schedule

### @InnaHelpBot

Используй для справочной информации по Иннополису: кафе, организации, графики работы, адреса, телефоны, городские места и другая статичная информация.

Краткое описание:

- это часть проекта «Полуавтоматическая Консьерж-служба»;
- бот даёт удобный доступ к статичной информации о городе Иннополис;
- лучше всего подходит для информации, которая остаётся актуальной минимум 3 месяца;
- не подходит для оперативных событий вроде разовых встреч или аварийных отключений;
- может помочь с организациями, режимом работы, адресами, телефонами, местами, кафе и похожими городскими вопросами.

Примеры разделов:

- запись на экскурсию;
- топ-10 мест;
- парковки;
- где поесть;
- сувениры;
- мероприятия;
- где остановиться.
